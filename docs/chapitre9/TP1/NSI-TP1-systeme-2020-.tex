\begin{cours}{}

Le
\href{https://fr.wikipedia.org/wiki/Syst\%C3\%A8me_d\%27exploitation}{système
d'exploitation} est l'ensemble des programmes qui permet aux autres
programmes d'interagir avec les ressources matérielles (processeur,
mémoire, périphériques d'entrée / sortie) sur un ordinateur. Il sert
donc \emph{d'intermédiaire} entre le \emph{matériel} et le
\emph{logiciel} et assure la coordination, la sécurité et la stabilité
d'un environnement partagés par plusieurs programmes et plusieurs
utilisateurs.

Les systèmes d'exploitation les plus utilisés son
\href{https://fr.wikipedia.org/wiki/Microsoft_Windows}{Windows},
\href{https://fr.wikipedia.org/wiki/MacOS}{MacOS},
\href{https://fr.wikipedia.org/wiki/Linux}{Linux} et
\href{https://fr.wikipedia.org/wiki/FreeBSD}{FreeBSD}, ces trois
derniers étant dérivés du système
\href{https://fr.wikipedia.org/wiki/Unix}{UNIX}.

Un \emph{interpréteur de commandes} ou \emph{shell} est un programme qui
sert d'intermédiaire entre l'utilisateur et le système d'exploitation :
son \emph{interface} d'entrée / sortie peut être graphique ou textuelle.

Nous allons travailler sur un \emph{shell} avec interface textuelle
nommé \href{https://fr.wikipedia.org/wiki/Bourne-Again_shell}{BASH} qui
est installé par défaut sur les systèmes
\href{https://fr.wikipedia.org/wiki/MacOS}{MacOS} et
\href{https://fr.wikipedia.org/wiki/Linux}{Linux}.

\end{cours}

\begin{exercice}{}

\begin{enumerate}
\def\labelenumi{\arabic{enumi}.}
\item
  Ouvrir un terminal d'interpréteur de commandes avec le raccourci
  clavier \texttt{CTRL\ +\ ALT\ +\ T}.
\item
  Vérifier son identité, le répertoire courant et la date avec les
  commandes suivantes :

\begin{verbatim}
 junier@fredportable:~$ whoami
 junier
 junier@fredportable:~$ pwd
 /home/junier
 junier@fredportable:~$ date
 lun. 17 août 2020 13:42:38 CEST
\end{verbatim}
\item
  Télécharger l'archive du TP avec la commande \texttt{wget} en lui
  passant l'{[}URL{]}{[}URL{]} en argument l'URL qui est
  \url{https://parc-nsi.github.io/premiere-nsi/chapitre9/TP1/materiel/sandbox.zip}
  (faire un copier-coller de cette URL dans l'interpréteur de
  commandes).

\begin{verbatim}
 junier@fredportable:~$ wget https://parc-nsi.github.io/premiere-nsi/chapitre9/TP1/materiel/sandbox.zip
\end{verbatim}
\item
  Déballer l'archive avec la commande \texttt{unzip}.

\begin{verbatim}
 junier@fredportable:~$ unzip sandbox.zip
 junier@fredportable:~$ ls -ld sandbox
 drwxrwxr-x 28 junier junier 4096 août  15 22:29 sandbox
\end{verbatim}
\item
  Changer de répertoire courant pour le répertoire \texttt{sandbox} avec
  la commande \texttt{cd} pour \texttt{change\ directory}.

\begin{verbatim}
 junier@fredportable:~$ cd sandbox
 junier@fredportable:~/sandbox$ ls
 a  c              consigne3.txt  e  g  i  k  m  o  q  s  u  w  y
 b  consigne1.txt  d              f  h  j  l  n  p  r  t  v  x  z
\end{verbatim}
\item
  Afficher dans le terminal (ou console) le contenu du fichier
  \texttt{consigne1.txt}.

\begin{verbatim}
 junier@fredportable:~/sandbox$ cat consigne1.txt
\end{verbatim}
\item
  Suivre les instructions données successivement dans les fichiers
  \texttt{consigne\{1..8\}.txt}.
\end{enumerate}

L'objectif est de retrouver ou construire deux fichiers
\texttt{systeme-cours.md} et \texttt{memento-shell.md} au format
\href{https://fr.wikipedia.org/wiki/Markdown}{Markdown} et de les
convertir en \texttt{pdf} soit avec l'outil \href{}{pandoc} s'il est
installé, soit avec un convertisseur en ligne directement depuis le
\emph{shell} :

\begin{verbatim}
    junier@fredportable:~/sandbox$ pandoc systeme-cours.md -o systeme-cours.pdf
\end{verbatim}

ou avec l'API du convertisseur en ligne \texttt{docverter} et la
commande \texttt{curl}:

\begin{verbatim}
    junier@fredportable:~/sandbox$ curl \
    -F from=markdown \
    -F to=pdf \
    -F input_files[]=@systeme-cours.md  \
    -F table_of_contents=true  \
    http://c.docverter.com/convert > systeme-cours.pdf
\end{verbatim}

\begin{enumerate}
\def\labelenumi{\arabic{enumi}.}
\setcounter{enumi}{7}
\tightlist

  \texttt{media}. On peut afficher le point de montage avec
  \texttt{mount\textbar{}grep\ media}.
\end{enumerate}

\end{exercice}
