\section{Web et Internet}

\begin{histoire}{}
En 1989, \href{https://fr.wikipedia.org/wiki/Tim_Berners-Lee}{Tim Berners-Lee}, qui s'occupe de la gestion des documents scientifiques au \href{https://home.cern/fr/science/computing/birth-web}{CERN},
propose avec le projet \textit{mesh} (filet) de faire traverser le réseau internet à \textbf{l'hypertexte} imaginé par \href{https://fr.wikipedia.org/wiki/Ted_Nelson}{Ted Nelson} en 1965. C'est la naissance du \textit{World Wide Web} : des liens hypertextes relient des  documents stockés sur des machines différentes et l'utilisateur d'une machine cliente peut naviguer de lien en lien à travers ce réseau logique d'informations qui se superpose au réseau physique d'internet.

Avec l'ouverture au public des standards du Web en 1993 et leur  standardisation organisée par le \href{https://www.w3.org/}{W3C}, qui assure la portabilité des pages Web sur toutes les machines,  le Web s'impose à partir du milieu des années 90 comme le service le plus populaire disponible sur Internet.
\end{histoire}



\section{Les fondements  du Web}

\subsection{Architecture client / serveur}

\begin{cours}{}
Comme la plupart des services d'Internet, le Web repose sur une \textbf{architecture client / serveur} : pour obtenir une page Web,  le  \textbf{client} la demande au \textbf{serveur} qui la détient. 

Le client et le serveur sont des logiciels et peuvent être installés sur n'importe quelle machine.

Le logiciel client s'appelle un \textbf{navigateur}, comme par exemple \textit{Firefox} ou \textit{Chrome}.
\end{cours}

\subsection{\texttt{URL} : l'adressage logique}

\begin{cours}{}

Les documents disponibles sont localisés sur le Web par une adresse logique, \href{https://fr.wikipedia.org/wiki/Uniform_Resource_Locator}{Uniform Ressource Locator}.

La syntaxe des URL est standardisée, par exemple décomposons :
\begin{center}
 \texttt{https://www.gnu.org/gnu/linux-and-gnu.fr.html} 
\end{center}


\begin{itemize}
	\item le protocole est \texttt{https} ;
	\item le nom de domaine sur Internet de la machine est \texttt{www.gnu.org}, le préfixe \texttt{www} étant le nom usuel du dossier public sur un serveur Web ;
	\item \texttt{gnu/linux-and-gnu.fr.html} est le chemin vers la ressource sur le serveur : le fichier \texttt{linux-and-gnu.fr.html} qui se trouve dans le dossier \texttt{gnu} lui-même dans le dossier public \texttt{www}. 
\end{itemize}

\end{cours}

\subsection{\texttt{HTTP} : le protocole}


\begin{cours}{}
Lors d'un échange Web, le \textbf{client} et le \textbf{serveur} dialoguent selon des règles fixées par un protocole standardisé, \href{https://fr.wikipedia.org/wiki/Hypertext_Transfer_Protocol}{HyperText Transfer Protocol}.

\begin{center}

	\includegraphics[scale=0.4]{schema-requete-http.jpg}
	
	{\small \textit{Source :} \url{https://wpmarmite.com/wordpress-https/}}
\end{center}
Si la page demandée n'est pas disponible, le serveur retourne le célèbre code d'erreur 404.

Nous étudierons plus en détail le protocole \texttt{HTPP} un peu plus tard dans l'année.

\end{cours}

\subsection{\texttt{HTML} : le langage}

\begin{cours}{}

Les \href{info.cern.ch/hypertext/WWW/TheProject.html}{premières pages Web} sont des \textbf{pages statiques}, \ie{} que la page retournée par le serveur ne dépend pas du client. Le premier langage du Web est \href{https://developer.mozilla.org/fr/docs/Glossaire/HTML}{HyperText Markup Language} qui est un langage de description de documents à l'aide de \textbf{balises}.  Un fichier écrit en \texttt{HTML} est un simple fichier texte qui est interprété par le navigateur pour obtenir un affichage. Pour éditer un fichier texte, on utilise un \textbf{éditeur de textes} comme \href{https://notepad-plus-plus.org/}{Notepad++}.

\texttt{HTML} permet de définir la structure d'un document en le découpant en \textbf{éléments} logiques délimités par une \textbf{<balise>} ouvrante et une \textbf{</balise>}  fermante. En particulier il permet de créer des liens hypertextes entre deux éléments d'un même document ou de documents différents.


L'agencement des \textbf{balises} doit respecter des règles de bonne formation : pour les balises imbriquées on procède comme pour les parenthèses.

\begin{lstlisting}[language=html,numbers=none,style=rond]
<div>  Ma partie

</div>
\end{lstlisting}
\end{center}
\end{minipage}


  
On donne ci-dessous la structure minimale d'un fichier \texttt{HTML} :

\begin{itemize}
	\item Une balise \texttt{<!DOCTYPE html>} qui  déclare le type du document.
	\item Une balise racine \texttt{<html>}.
	\item Une balise d'entête \texttt{<head>} avec des meta-informations qui ne sont pas affichées.
	\item Une balise de corps \texttt{<body>} qui contient tout le contenu affiché.
	\item Des commentaires dans une balise spéciale \texttt{<!--  ici le commentaire --> }.
\end{itemize}


\end{cours}

\section{Langages du Web}

\subsection{Principe de séparation}

\begin{cours}{}

Une  page Web est un document de structure non linéaire (liens hypertextes), avec des contenus riches (images, video) et des interactions possibles côté client (navigateur) ou côté serveur (accès à une base de données).

On distingue les pages Web \textbf{statiques} dont le contenu ne dépend que de l'utilisateur et de ses interactions, des pages Web \textbf{dynamiques}. 

Un \textbf{principe de séparation} des fonctionnalités est appliqué dans  la  réalisation d'une page Web :

\begin{itemize}
	
	\item La structure du document est définie par le langage \texttt{HTML}. Pour les pages Web dynamiques, le code \texttt{HTML} peut être généré par le serveur et envoyé au client à sa demande.
	\item L'apparence et la disposition des éléments sont définies par le langage \texttt{CSS}.
	\item La gestion des événements utilisateurs (clics ...) est assurée par le langage \texttt{Javascript} qui peut modifier la structure du document \texttt{HTML} dans le navigateur du client.

\end{itemize}


\end{cours}


\subsection{HTML pour la structure}

\begin{cours}{}
Le langage \texttt{HTML} permet de définir la structure d'une page Web.

Il existe deux grandes catégories de balises :

\begin{itemize}
	\item \textbf{balises en-ligne} qui ne sont pas affichées après un saut de ligne, se juxtaposent  et occupent juste la place nécessaire à leur contenu en largeur. Certaines de ces balises sont vides, elles n'ont pas de contenu et donc pas de balise fermante.
	\item \textbf{balises blocs} qui sont  affichées après un saut de ligne, se superposent et occupent toute la place possible en largeur. 
\end{itemize}

Certaines balises ont une \textbf{sémantique} prédéfinie : \texttt{<article>}, \texttt{<section>}, très utile pour l'indexation de la page par un moteur de recherche. D'autres balises comme \texttt{<span>} (en-ligne)  ou \texttt{<div>} (bloc) sont neutres et personnalisables par le développeur.
\end{cours}


\begin{methode}{}

\begin{itemize}

\item Les principales balises de type \textbf{en-ligne} qu'il faut connaître :


\begin{tabularx}{\linewidth}{|c|c|>{\ttfamily}X|}
\hline 
Balise  & Sémantique & Syntaxe \\ 
\hline 
<a> & lien hypertexte & <a href="https://site.org/index.html">Lien</a> \\ 
\hline 
<br> & saut de ligne & <br>  \\ 
\hline 
<img> & insertion d'image &  	<img src="https://linux.org/Tux.png" alt="tux"> \\ 
\hline 
<strong> & texte très important  & <strong>important</strong> \\ 
\hline 
<em> & texte important & <em>important</em> \\ 
\hline 
<span> & balise neutre & <span>contenu</span> \\ 
\hline 
\end{tabularx} 

En général, le contenu de la balise <strong> est en gras et celui de la balise <em> en italique.

\item Les principales balises de type \textbf{bloc} qu'il faut connaître



\begin{tabularx}{\linewidth}{|c|c|>{\ttfamily}X|}
\hline 
Balise  & Sémantique & Syntaxe \\ 
\hline 
<p> & paragraphe & <p>mon paragraphe. </p> \\ 
\hline 
<li> & item de liste& <li>item</li>  \\ 
\hline 
<ol> & liste ordonnée & <ol><li>lundi</li><li>mardi</li></ol>  \\ 
\hline 
<ul> & liste non ordonnée & <ul><li>basket</li><li>short</li></ul>  \\ 
\hline

\end{itemize}


\end{methode}


\subsection{CSS pour la mise en forme}

\begin{cours}{}
Le langage \texttt{CSS} permet de définir l'apparence et la disposition des éléments \texttt{HTML}.

Pour respecter le principe de séparation, les \textbf{définitions de style CSS} sont regroupées dans un fichier texte d'extension \texttt{.css} qui est  relié au fichier source \texttt{HTML} par une balise \texttt{link} placée dans l'entête du document. 

On parle souvent de \textbf{feuille de style CSS}.


\begin{lstlisting}[style=rond]
<link href="style.css" rel="stylesheet" type="text/css" />
\end{lstlisting}


On peut aussi définir des règles \texttt{CSS} directement dans let fichier  \texttt{HTML} dans un attribut de balise \texttt{style} ou dans l'entête du document à l'intérieur d'une balise \texttt{<style>}. Si un élément n'a pas de règle de style particulière il applique celle de son plus proche  parent.  Si un élément est ciblé par plusieurs règles,  la plus locale s'applique : attribut \texttt{style}  > balise \texttt{<style>} > fichier {.css}.

Le principe  d'une règle de style \texttt{CSS} est  d'appliquer un ensemble de définitions (propriété : "valeur") à un ensemble d'éléments \texttt{HTML} défini par un \textbf{sélecteur}.

\begin{lstlisting}[style=rond]
p { 
	propriete1 : "valeur";
	propriete2 : "valeur";
}

#monidentifiant {
	propriete1 : "valeur";
}

.uneclasse {
	propriete2 : "valeur";
}
\end{lstlisting}

Un \textbf{sélecteur} CSS peut être :

\begin{itemize}
	\item Le type d'un élément \texttt{HTML} par exemple \texttt{p} pour cibler tous les paragraphes ;
	\item Un \textbf{identifiant }avec le préfixe \verb+#+  pour cibler l'unique élément   qui a été marqué par cet identifiant dans le document \texttt{HTML} avec la syntaxe \texttt{<element id="monidentifiant">}.
		\item Une \textbf{classe} avec le préfixe .  pour cibler un ensemble d'éléments, pas forcément de même type,   qui ont  été marqués par cette classe dans le document \texttt{HTML} avec la syntaxe \texttt{<element class="uneclasse">}.
		\item Une combinaison de sélecteurs, voir \url{https://www.w3schools.com/Css/css_combinators.asp}.

\end{itemize}

Voir la page \url{https://www.w3schools.com/Css/css_intro.asp} pour la diversité des actions possibles avec une   feuille de style \texttt{CSS} sur un même code source \texttt{HTML}.

On donne ci-dessous un exemple de feuille de style \texttt{CSS} associée à un fichier source \texttt{HTML} avec l'affichage obtenu.



\end{cours}
